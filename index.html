Project objective

This project segments customers by Age, Annual Income, and Spending Score to reveal meaningful behavior groups. The app compares two clustering approaches, shows the difference visually, and translates the math into actionable business insights.

Why k-Means

k-Means uses centroids, which are average points. It is fast and excellent for clean, compact datasets, which is why it is often the first algorithm taught for customer segmentation.

Why k-Medoids

k-Medoids chooses actual data points as centers. That makes it slower, but more resistant to unusual values and easier to explain because each segment has a real representative customer.

Architecture summary

What the app is doing behind the scenes.

Data layer CSV parsing + smart column mapping Handles the default Mall Customers dataset and alternate uploads with different header names.
Analytics layer k-Means, k-Medoids, elbow, silhouette Everything runs client-side in the browser, so the project stays a true single-file web app.
Persistence layer Firestore with local fallback Saved runs are written to Firestore when rules allow it, otherwise they remain in local browser history without breaking the app.

Important Firebase note

Because this is a single HTML file, the Firebase web config is used directly in the browser.

Your Firebase config is safe to place in the frontend, but Firestore access is controlled by your Firebase security rules. If writes are denied, the app still works and saves run history locally until you relax the rules or add authentication later.
